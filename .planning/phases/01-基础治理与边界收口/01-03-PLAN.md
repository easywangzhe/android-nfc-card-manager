# Plan 03: 路由权限与页面状态表达统一

> 兼容性副本：为满足 GSD `*-PLAN.md` 扫描规则而补建，保留原文件 `01-PLAN-03.md` 以避免破坏已有引用。

## Objective

在已有导航守卫基础上补齐执行层权限兜底，并把 Phase 1 的共享阶段词汇和真实性模型接到页面文案与状态卡片上，让用户能看懂“能不能做、现在做到哪、结果是真是假”。

## Requirements

- SHELL-03
- SHELL-04
- SHELL-05
- FLOW-01
- FLOW-03

## Scope

- 在执行入口增加角色校验，避免误入页面后仍能继续操作
- 页面状态展示统一映射为等待、扫描中、处理中、成功、失败
- demo-only / 未验证 / 未支持状态统一接入能力或结果展示
- 保持当前导航结构不重构，只补 Phase 1 所需边界和表达一致性

## Files In Scope

- `app/src/main/java/com/opencode/nfccardmanager/navigation/AppNavGraph.kt`
- `app/src/main/java/com/opencode/nfccardmanager/core/security/SecurityManager.kt`

- `app/src/main/java/com/opencode/nfccardmanager/feature/unlock/`

## Tasks

### 1-03-01 执行层权限兜底

- 在各 ViewModel 或启动入口增加角色校验辅助函数
- 即使通过深链或异常导航进入页面，也不能开始无权 NFC 操作
- 失败时返回明确拒绝原因，而不是静默无响应
- 验证: `./gradlew test assembleDebug`

### 1-03-02 页面阶段与真实性文案统一

- 读卡、写卡、格式化、锁卡、解锁页接入共享状态展示文案
- 演示按钮、演示结果和未验证能力必须显式标注，不能伪装成真实成功
- 调整页面初始提示，使用户先看到等待开始而不是扫描已启动
- 验证: `./gradlew assembleDebug`

## Wave Plan

- Wave 1: 执行层权限辅助函数与拒绝提示
- Wave 2: 状态卡片统一映射
- Wave 3: demo-only 与未验证能力标签补齐

## Exit Criteria

- 路由守卫和执行守卫同时成立
- 任一流程都能用一致词汇表达当前阶段
- 用户可直接区分真实支持、未验证、仅演示、未支持

## Risks

- 若状态统一只改文案、不改触发时机，仍会造成“文案正确但行为错误”
- 若 demo-only 标识只出现在详情文字末尾，仍会误导用户
