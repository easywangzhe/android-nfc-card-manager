# Plan 01: 共享治理契约落地

## Objective

为所有 NFC 流程建立可复用的共享契约，先统一“阶段语义”“能力真实性”和“单会话仲裁”三类基础模型，避免后续页面各自维护状态和文案。

## Requirements

- SHELL-05
- FLOW-02
- FLOW-03

## Scope

- 新增共享流程阶段显示模型，映射等待、扫描中、处理中、成功、失败
- 新增共享能力真实性模型，统一支持、未验证、仅演示、未支持四种语义
- 新增单一 NFC 会话协调器，记录当前活动操作类型、会话 token 和 owner
- 补齐 Wave 0 单元测试骨架，覆盖阶段/真实性映射和会话仲裁

## Files In Scope

- `app/src/main/java/com/opencode/nfccardmanager/core/nfc/`
- `app/src/main/java/com/opencode/nfccardmanager/core/nfc/model/`
- `app/src/test/java/com/opencode/nfccardmanager/core/nfc/`

## Tasks

### 1-01-01 统一阶段与真实性契约

- 新增共享阶段显示模型与映射函数
- 新增真实性枚举与显示文案/视觉语义映射
- 保持各 feature 原有 stage 枚举不大改，只做映射层收口
- 验证: `./gradlew test --tests "*NfcFlowContract*"`

### 1-01-02 新增单会话协调器

- 在 `core/nfc/` 提供单例或 object 级协调器
- 支持 `requestStart`、`release`、`isOwner`、`activeSession` 等最小接口
- 拒绝并发启动，且旧 token/非 owner 释放不能误清空当前会话
- 验证: `./gradlew test --tests "*NfcSessionCoordinator*"`

## Wave Plan

- Wave 0: 建测试文件与最小可测共享模型
- Wave 1: 落地共享阶段/真实性模型
- Wave 2: 落地会话协调器并补测试

## Exit Criteria

- 共享阶段词汇可被读卡、写卡、格式化、锁卡、解锁复用
- demo-only 与真实支持状态不再只能靠自由文本表达
- 任一 NFC 流程都可通过同一协调器判断是否允许开启会话

## Risks

- 若在此计划中直接替换全部 feature stage 枚举，会放大改动范围
- 若协调器接口设计过重，会迫使页面层一起大改

## Notes

- 优先做“共享模型 + 映射层”，不在本计划内做页面视觉重写
