# Plan 02: NFC 流程显式启动与会话收口

## Objective

把读卡、写卡、格式化、锁卡、解锁五类 NFC 流程改成“用户显式开始后才进入扫描”，并通过共享协调器保证同一时间只存在一个生效会话。

## Requirements

- FLOW-01
- FLOW-02
- FLOW-03

## Scope

- 清理页面进入即自动开扫的副作用
- 所有 ReaderMode 开启前都先申请共享会话
- 页面离开、成功、失败、超时后都正确释放会话
- 保持原有底层 `TagParser`、`NdefWriter`、`NdefFormatter`、`NdefLocker`、`UnlockExecutor` 不做重写

## Files In Scope

- `app/src/main/java/com/opencode/nfccardmanager/feature/scan/ScanScreen.kt`
- `app/src/main/java/com/opencode/nfccardmanager/feature/write/WriteEditorScreen.kt`
- `app/src/main/java/com/opencode/nfccardmanager/feature/format/FormatCardScreen.kt`
- `app/src/main/java/com/opencode/nfccardmanager/feature/lock/LockRiskScreen.kt`
- `app/src/main/java/com/opencode/nfccardmanager/feature/unlock/UnlockVerifyScreen.kt`
- `app/src/main/java/com/opencode/nfccardmanager/core/nfc/NfcSessionManager.kt`

## Tasks

### 1-02-01 显式开始与单会话接入

- 读卡页移除 `DisposableEffect` 初始自动开扫
- 格式化页移除 `LaunchedEffect(Unit)` 自动 `start()`
- 写卡、锁卡、解锁页在进入扫描前统一走显式开始动作
- 所有页面接入共享会话协调器，启动失败时给出已存在活动会话提示
- 验证: `./gradlew test assembleDebug`

### 1-02-02 扫描中/处理中状态与超时释放修正

- 扫描态只在成功申请会话并开启 ReaderMode 后生效
- 处理中时阻止重复开始同一流程
- 成功、失败、超时、离开页面时统一释放会话
- 验证: `./gradlew assembleDebug`

## Wave Plan

- Wave 1: 读卡与格式化先改成显式开始
- Wave 2: 写卡、锁卡、解锁接入协调器
- Wave 3: 收口超时、离开页面和重复触发释放逻辑

## Exit Criteria

- 所有 NFC 页面初始均处于等待态
- 用户不点击开始时不会触发 ReaderMode
- 快速切页或回退不会保留幽灵扫描会话

## Risks

- Compose 生命周期与 ReaderMode 绑定紧密，回收时机需要保守处理
- 若只拦截 UI 按钮、不拦截底层会话申请，仍可能留下重复执行窗口
