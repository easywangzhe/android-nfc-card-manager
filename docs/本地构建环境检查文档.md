# Android NFC 卡片管理 App 本地构建环境检查文档

## 1. 文档目的
用于指导开发者在本地完成 Android 项目的构建环境准备、问题自查与基础验证。

## 2. 项目构建前提

当前项目已包含：

- Gradle Wrapper
- Android 应用模块
- Compose 基础工程
- 本地文档与代码结构

本地要成功构建，还需要开发机具备：

1. JDK 17
2. Android SDK
3. 正确配置的 `JAVA_HOME`
4. 正确配置的 Android SDK 路径

---

## 3. 必备环境检查

## 3.1 检查 Java

执行：

```bash
java -version
```

期望：
- 可正常输出 Java 版本
- 版本建议为 `17`

若失败：
- 说明未安装 JDK
- 或 `JAVA_HOME` / PATH 未正确配置

---

## 3.2 检查 JAVA_HOME

执行：

```bash
echo $JAVA_HOME
```

期望：
- 指向 JDK 17 安装目录

macOS 常见示例：

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export PATH="$JAVA_HOME/bin:$PATH"
```

---

## 3.3 检查 Android SDK

确认以下目录存在其一：

- `$ANDROID_HOME`
- `$ANDROID_SDK_ROOT`
- Android Studio 默认 SDK 目录

macOS 常见位置：

```text
~/Library/Android/sdk
```

---

## 3.4 配置 local.properties

在项目根目录创建：

```text
local.properties
```

内容示例：

```properties
sdk.dir=/Users/你的用户名/Library/Android/sdk
```

注意：
- 路径必须是你机器上的真实 SDK 路径
- `local.properties` 不应提交到版本库

---

## 4. 推荐安装组件

在 Android Studio SDK Manager 中确认安装：

1. Android SDK Platform 35
2. Android SDK Build-Tools
3. Android SDK Platform-Tools
4. Android Emulator（如需）
5. Android SDK Command-line Tools（建议）

---

## 5. 构建命令

进入项目根目录：

```bash
cd android-nfc-card-manager
```

### 5.1 查看任务

```bash
./gradlew tasks
```

### 5.2 构建 Debug 包

```bash
./gradlew assembleDebug
```

### 5.3 运行单元测试

```bash
./gradlew test
```

### 5.4 查看依赖或诊断

```bash
./gradlew dependencies
```

---

## 6. 常见问题排查

## 6.1 `Unable to locate a Java Runtime`
原因：
- 未安装 JDK
- 或系统找不到 Java

处理：
- 安装 JDK 17
- 配置 `JAVA_HOME`
- 重启终端后重试

---

## 6.2 `SDK location not found`
原因：
- 未配置 Android SDK 路径

处理：
- 创建 `local.properties`
- 填写正确的 `sdk.dir`

---

## 6.3 Gradle 下载失败
原因：
- 网络问题
- 代理问题

处理：
- 检查网络
- 必要时配置代理
- 重试 `./gradlew tasks`

---

## 6.4 Android 依赖解析失败
原因：
- Google Maven / Maven Central 无法访问
- 构建插件版本与环境不兼容

处理：
- 检查网络
- 更新 Android Studio 和 SDK
- 确保 JDK 版本正确

---

## 7. 构建成功标准

满足以下任一可视为基础环境通过：

1. `./gradlew tasks` 成功
2. `./gradlew assembleDebug` 成功
3. `./gradlew test` 成功

建议最低标准：
- 能成功执行 `./gradlew assembleDebug`
