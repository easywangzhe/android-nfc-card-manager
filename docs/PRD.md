# Android NFC 卡片管理 App PRD（V1）

## 1. 文档信息
- 产品名称：NFC 卡片管理 App
- 版本：V1.0 PRD
- 平台：Android
- 目标：支持指定 NFC 卡型的读卡、写卡、锁卡、解锁与操作审计

## 2. 产品背景
企业在标签初始化、资产标识、设备绑定、生产履历写入等场景中，需要通过 Android 手机对 NFC 卡/标签进行快速管理。

当前痛点：
- 读写依赖专用设备，使用门槛高
- 写卡缺乏校验，误操作风险高
- 锁卡/解锁流程不规范，容易造成不可逆错误
- 缺少日志与责任追踪

## 3. 产品目标

### 3.1 核心目标
1. 支持目标 NFC 卡片的读取与写入
2. 支持锁卡与可解锁卡的解锁
3. 降低误写、误锁风险
4. 提供完整操作记录

### 3.2 V1 成功标准
- 可稳定完成目标卡型的读写
- 可识别卡片是否已锁/是否可写
- 锁卡流程具备高风险确认
- 所有关键操作可追溯

## 4. 产品范围

### 4.1 V1 支持范围
- 支持 Android NFC 手机
- 支持指定 NFC 标签/卡型
- 支持：读卡、写卡、锁卡、解锁、操作日志、模板写卡

### 4.2 V1 不支持范围
- 不承诺支持所有 NFC 卡型
- 不支持银行卡/公交卡/未知加密门禁卡破解
- 不支持非法复制、绕过密钥、修改受保护安全区
- 不支持 iOS

## 5. 用户角色

### 5.1 普通操作员
- 可读卡
- 可写卡
- 可查看结果与历史
- 不可执行高风险锁卡/解锁，或需审批

### 5.2 管理员
- 可执行锁卡/解锁
- 可管理模板
- 可查看全部审计日志
- 可配置权限策略

## 6. 典型使用场景
1. 运维人员读取设备标签内容
2. 工厂人员批量初始化标签数据
3. 管理员对正式投入使用的标签执行锁定
4. 管理员对授权标签执行解锁后重新写入
5. 内审查看某张卡的修改记录

## 7. 功能需求

### 7.1 首页/工作台
- 展示读卡、写卡、锁卡、解锁入口
- 展示 NFC 状态、最近操作记录

### 7.2 读卡
- 自动识别卡型
- 自动读取 UID、卡类型、数据内容、锁定状态
- 支持结构化视图与 Hex 视图
- 支持保存结果到历史

### 7.3 写卡
- 支持模板、历史记录、手动输入作为数据源
- 支持格式校验、长度校验、写前预览
- 写后自动回读验证

### 7.4 锁卡
- 支持永久只读锁定
- 支持密码/密钥保护锁定（视卡型而定）
- 自动检测卡型是否支持锁定
- 风险提示、二次确认、管理员身份验证

### 7.5 解锁
- 仅对支持密码/密钥机制的卡开放
- 支持凭据输入、管理员验证、结果确认

### 7.6 模板管理
- 模板列表、新建、编辑、删除、版本标识

### 7.7 操作日志/审计
- 记录操作时间、操作者、设备信息、卡 UID、卡类型、操作类型、结果、错误原因、变更摘要

## 9. 非功能需求
- 性能：识别后尽快完成读卡反馈，UI 不得卡死
- 安全：密钥不明文展示，高风险操作需管理员权限
- 兼容性：以已验证机型 + 已验证卡型为准
- 可用性：普通操作 3 步内可进入扫描

## 10. 风险与约束
- 不同卡型锁卡/解锁机制差异大
- 部分卡锁定不可逆
- 部分 Android 手机不支持特定卡型
- 密钥体系设计不当会引发安全问题
